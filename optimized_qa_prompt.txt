# Improved Prompt

Generate complete, executable Playwright TypeScript test cases for: {input}

**Requirements:**
1. **Code Structure:**
   - Implement Page Object Model (POM) pattern with separate page class files
   - Write complete, runnable test code (not pseudocode)
   - Include all necessary imports and setup

2. **Test Coverage:**
   - Write at least ONE happy path test case (successful flow)
   - Write at least ONE negative test case (error/validation scenario)
   - Include all intermediate steps (not just final assertions)

3. **Selectors:**
   - Use robust selectors: `getByRole()`, `getByLabel()`, `getByTestId()`, `getByPlaceholder()`
   - Avoid fragile selectors like `#id` or `.class` unless absolutely necessary
   - Add `data-testid` attributes where needed

4. **Assertions:**
   - Assert page navigation/URL changes
   - Assert element visibility and state changes
   - Assert success/error messages
   - Assert data persistence where applicable

5. **Specificity:**
   - Include ALL form fields and interactions (dates, selections, confirmations)
   - Specify exact test data values
   - Include modal/dialog interactions and assertions
   - Add wait conditions for dynamic content

**Output Format:**
```typescript
// page.ts - Page Object Model class
// test.spec.ts - Test cases with happy path and negative scenarios
```
